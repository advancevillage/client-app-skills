# App Rive Skill

Use this skill when a Flutter App feature involves Rive animation design, Rive Editor operation guidance, or `.riv` asset integration.

---

## 适用场景

- 设计稿里有互动动画、状态切换、按钮反馈、插画动效、加载态、空态反馈。
- 从 UI/UX 设计稿（Pencil）中已拿到组件信息，需要转写为 Rive Editor 操作。
- 用户明确表示自己是 Rive 新手，需要一步一步指导。

必须：
- 先遵守 `app-spec` 分层边界，再应用本 skill。
- 图片组件先问：`这个图片组件是否需要制作为 Rive 动画交互？`
- 如果设计稿信息已知，必须直接写尺寸、图层、连线、时长，不能只说“按设计稿来做”。

禁止：
- 默认所有图片都转成 Rive。
- 只给 Flutter 接入代码，不说明 `.riv` 如何制作。
- 用概念解释替代具体操作。

---

## Pencil接入法

当信息来源是 `Pencil` 时，默认按以下规则处理。

必须：
- 先从 `Pencil` 查询组件类型、尺寸、层级、文本、图片和交互意图。
- 如果查询结果显示组件本质是图片，先问：
  `这个图片组件是否需要制作为 Rive 动画交互？`
- 如果 `Pencil` 已给出明确尺寸，回答里必须直接写数值，不能写“按设计稿尺寸”。
- 如果 `Pencil` 已给出层级或组件结构，回答里必须直接转写为 `画布内容层`。
- 如果 `Pencil` 已给出状态或交互说明，回答里必须直接转写为 `状态连线路` 和 `动画时间表`。

默认转写：
- `Pencil` 尺寸信息 -> `画布尺寸图`
- `Pencil` 图层/组件结构 -> `画布内容层`
- `Pencil` 状态说明/交互说明 -> `状态连线路`
- `Pencil` 节奏说明/反馈说明 -> `动画时间表`

禁止：
- 已有 `Pencil` 信息时，仍输出泛化建议。
- 忽略 `Pencil` 中已存在的尺寸、层级、状态信息，自行猜测。

## 接话模板

先说：

`已从 UI/UX 设计稿获取组件信息，下面直接转写为 Rive Editor 可执行参数。`

如果是图片组件，先问：

`这个图片组件是否需要制作为 Rive 动画交互？`

如果用户选择是，接着说：

`下面我会直接给出画布尺寸、画布内要做的内容、状态机连线方式、每段动画时长，以及在 Rive Editor 中的分步操作。`

如果用户是新手，接着说：

`我会按 Rive 新手方式说明，每一步都包含操作、完成后你会看到什么，以及下一步前要检查什么。`

---

## 是否转Rive

先给结论，再给原因。

适合：
- 图标微动效
- 按钮点击反馈
- 插画分层动画
- 成功、失败、空态切换

不适合：
- 照片类位图
- 强纹理整图
- 无法拆层的单图

不适合时必须建议：
- 静态图片 + Flutter 原生动效
- 重绘为 SVG/AI 分层素材后再导入 Rive

---

## 画布尺寸图

必须写：
- Artboard 宽度
- Artboard 高度
- 是否 1:1 建立
- 是否预留安全留白

示例：

```text
画布尺寸图
- Artboard：343 x 120
- 建立方式：1:1
- 安全留白：上下各 8px
```

---

## 画布内容层

必须写：
- 画布里有哪些层
- 哪些是导入素材
- 哪些要在 Rive 中补绘
- 哪些参与动画
- 哪些保持静态

示例：

```text
画布内容层
- background：底板，静态
- icon_main：主图标，参与缩放和位移
- glow_ring：高光环，参与透明度变化
- shadow：阴影层，参与轻微缩放
- label_text：文案层，静态
```

---

## 状态连线路

必须写：
- 默认状态
- 动画状态
- Inputs
- 每条连线怎么接
- 哪条自动返回

示例：

```text
状态连线路
- 默认状态：idle
- Inputs：tap, isSuccess
- idle -> press：Trigger `tap`
- press -> success：Boolean `isSuccess == true`
- press -> idle：动画播完自动返回
- success -> idle：动画播完自动返回
```

---

## 动画时间表

必须写：
- 每个状态时长
- 关键帧时间点
- 是否循环
- 总反馈时长

示例：

```text
动画时间表
- idle：1200ms，循环
- press：180ms，关键帧 0 / 90 / 180ms
- success：600ms，关键帧 0 / 200 / 400 / 600ms
- 总反馈：不超过 800ms
```

---

## 操作步骤表

必须按步骤输出，每一步都包含：
- 操作
- 完成后你应该看到
- 下一步前检查

默认步骤：
1. 新建 Artboard
2. 导入或补绘内容
3. 创建 Timeline
4. 创建 State Machine
5. 预览与修正
6. 导出交付

Step 1 必须写清：
- 在 Rive Editor 里如何新建文件
- Artboard 名称
- Artboard 尺寸

Step 2 必须写清：
- 画布里具体放哪些层
- 哪些导入，哪些补绘
- 锚点和层级怎么检查

Step 3 必须写清：
- 每段 Timeline 名称
- 每段时长
- 关键帧时间点

Step 4 必须写清：
- State Machine 名称
- Inputs 名称
- 每条线的起点、终点、条件

---

## 导出接入项

必须列出：
- `.riv` 文件名
- Artboard 名称
- Artboard 尺寸
- State Machine 名称
- Inputs 名称
- 动画时长表

Flutter 侧只允许：
- 在 `presentation/` 层加载和驱动 Rive
- Provider/Notifier 负责状态和触发

禁止：
- `domain/`、`data/`、Repository、DTO、Entity 依赖 Rive 类型

---

## 后续调整项

默认输出 5 到 7 条：

1. 补全设计稿到 Rive 的映射信息：尺寸、图层、动画目标、交互触发。
2. 固定 Artboard 宽高和安全留白，避免回弹或缩放时裁切。
3. 明确每一层在画布中做什么，哪些参与动画，哪些只保留静态。
4. 逐条列出状态机连线，写清触发条件和自动返回规则。
5. 给出每段动画时长和关键帧时间点，控制总反馈节奏。
6. 预览检查裁切、锚点、遮挡、连线条件和重复触发表现。
7. 导出时补齐 `.riv`、尺寸、状态机、Inputs、时长清单。
